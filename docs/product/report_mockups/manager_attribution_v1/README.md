# Manager Mandate Attribution v1 Report Mockups

These local product-review mockups show manager implementation against manager mandate benchmarks.
They keep manager implementation separate from advisor-level policy effects.
The Manager Driver Attribution Matrix is the all-manager component view; selected-manager detail is its drill-down.
They are not wired into Advisor Preview, Populate, Present, generated reports, Docker, deployment, live data, external data, or production reporting.

## Gated Or Deferred

- Full Manager-by-Manager Driver Detail with Position Drilldown (Gated): Requires approved position-level driver inputs and drilldown design.
- Timing Attribution (Unavailable): Unavailable because clean timing inputs and an approved method are absent.
- Dollar P&L Attribution (Gated): Gated unless reliable beginning portfolio value exists.
- Production Client Manager Attribution (Gated): Gated on approved real manager returns and mandate benchmarks.
- Blended / All-In Attribution (Deferred): Deferred so advisor policy and manager implementation remain separate.
- Position-Level Manager Attribution (Gated): Requires approved position-level attribution inputs and review design.
