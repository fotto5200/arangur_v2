# Imputed Current Allocation Baseline

Imputed-current mode accepts current manager weights as the baseline and suppresses drift attribution.

## Key Metrics

- **Baseline weights:** Current weights
- **Drift status:** Suppressed
- **Use case:** Quick start

## Imputed-Current Baseline Use

| Topic | Meaning |
| --- | --- |
| Mode | Current manager weights are accepted as the baseline. |
| What it enables | Manager and mandate review can start before explicit targets are entered. |
| What it suppresses | Target-versus-actual drift attribution is suppressed. |
| When to use | Use for quick-start review when documented policy targets are missing. |
| Limit | It does not prove the current allocation is ideal. |

## Caveats

- Replace this baseline with explicit policy targets when policy documentation is available.
